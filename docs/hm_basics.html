1. Introduction


Histogram model can be used for data analysis and information retrieval based on the number of occurrences of elements that make up data instances. The basic unit of computation here is a histogram.

Next in this paper different types of histograms will be discussed, and then the histogram algebraic system will be introduced that is the main component of the histogram model. Finally, some simple examples of use of the histogram model will be considered.


2. Histograms


2.1. Basic definitions


Each data instance (e.g. document, image etc.), ddd, is supposed to consist of elements of the universal set, UUU.

In the histogram model two types of histograms are distinguished: a histogram of data instance, H(d)H(d)H(d), and histogram of elements given a data instance, H(X∣d)H(X \mid d)H(X∣d).

Histogram of a data instance, H(d)H(d)H(d), describes the number of occurrences of elements of the universal set, UUU. So, each element of histogram corresponds to a single element of the universal set, x∈Ux \in Ux∈U.

Histogram of elements given a data instance, H(X∣d)H(X \mid d)H(X∣d), is a histogram of a data instance that is limited to elements of X⊆UX \subseteq UX⊆U, and other elements are disregarded. When XXX corresponds to UUU, H(U∣d)H(U \mid d)H(U∣d) is equal to H(d)H(d)H(d).

2.2. Histogram of data instance


There are multiple notations for histograms [2]. For example, we can consider a histogram as a union of elements:

where h(x∣d)/xh(x \mid d)/xh(x∣d)/x denotes existence of element xxx and its occurrence value, h(x∣d)h(x \mid d)h(x∣d), in document ddd.

Alternatively, we can represent a histogram using an ordered list

H(d)=(h(e1),⋯ ,h(en)∣h(x)=h(x∣d),0≤h(x)≤N(d),∑x∈Uh(x)=N(d)).H(d)=\left( h(e_1), \cdots, h(e_n) \mid h(x)=h(x \mid d), 0 \leq h(x) \leq N(d), \sum_{x\in U}h(x) = N(d) \right).H(d)=(h(e1​),⋯,h(en​)∣h(x)=h(x∣d),0≤h(x)≤N(d),x∈U∑​h(x)=N(d)).

It's often helpful to use a normalized form, where each element xxx of a histogram calculated as

h(x∣d)=n(x∣d)N(d),h(x \mid d) = \frac{n(x \mid d)}{N(d)},h(x∣d)=N(d)n(x∣d)​,

So, the normalized histogram can be written as

H(d)=(h(e1),⋯ ,h(en)∣h(x)=h(x∣d),0≤h(x)≤1,∑x∈Uh(x)=1),H(d)=\left( h(e_1), \cdots, h(e_n) \mid h(x)=h(x \mid d), 0 \leq h(x) \leq 1, \sum_{x\in U}h(x) = 1 \right),H(d)=(h(e1​),⋯,h(en​)∣h(x)=h(x∣d),0≤h(x)≤1,x∈U∑​h(x)=1),

where h(x∣d)h(x \mid d)h(x∣d) refers to the normalized form of the number of occurrences.

3.1. Basic definition


As any algebraic system, the histogram one consists of a carrier set and a set of operations. Here HGH^GHG is used as a carrier set, and assumed to be in the binary form (note that for fuzzy elements, for example, the binary form is not applicable and must be used the general one as shown in [5]).

3.2. Rules of usage


Set and logical operations have different bases, and the algebraic system imposes constraints on use of the operations in expressions. So, the basic rule is the following:

Set operations should be applied before logic operations.

For example,

(((H1∩H2)∖(H3∪H4))∨˙H5)∧H6.((( H_1 \cap H_2 ) \setminus ( H_3 \cup H_4 )) \dot \lor H_5) \land H_6.(((H1​∩H2​)∖(H3​∪H4​))∨˙H5​)∧H6​.

Some combinations that break the basic rule should be transformed to be more reasonable in the sense of reflecting meaning, rather than mathematics, e.g.

(X1∨˙X2)∖X3→(X1∖X3)∨˙(X2∖X3), ( X_1 \dot \lor X_2 ) \setminus X_3 \to ( X_1 \setminus X_3) \dot \lor (X_2 \setminus X_3 ), (X1​∨˙X2​)∖X3​→(X1​∖X3​)∨˙(X2​∖X3​),
(X1∧X2)∖X3→(X1∖X3)∧(X2∖X3), ( X_1 \land X_2 ) \setminus X_3 \to ( X_1 \setminus X_3) \land (X_2 \setminus X_3 ), (X1​∧X2​)∖X3​→(X1​∖X3​)∧(X2​∖X3​),

3.3. Queries


As mentioned in the introduction, the histogram model can be applied for information retrieval [3,4,6]. There are two basic types of queries that can be used: expressions and samples.


Expressions as a query


In this case, a query can be specified by elements of EGE^GEG and operations of the histogram model in the following way

q=Expression(X1,X2,⋯ ,Xm). q = \text{Expression}(X_1, X_2, \cdots, X_m). q=Expression(X1​,X2​,⋯,Xm​).

To calculate a value of similarity between the query and data instances, it's necessary to evaluate the histogram expression and sum the values of the resulting histogram:

sim(q,d)=∑x∈UExpression(H1,H2,⋯ ,Hm;O)=ΣH. \text{sim}(q, d) = \sum_{x\in U}\text{Expression}(H_1, H_2, \cdots, H_m;O)={}^\Sigma H.sim(q,d)=x∈U∑​Expression(H1​,H2​,⋯,Hm​;O)=ΣH.

Generally, when we use complex expressions, resulting histograms may not reflect really the number of occurrences of expressions given a data instance. Think of, for example, the AND operation. It returns one of the operands, H1H_1H1​ or H2H_2H2​. Here the result is just based on the number of occurrences of elements X1X_1X1​ and X2X_2X2​ in ddd. So the term "value of expression" is more appropriate instead of "number of occurrences of expression". But when we talk about a histogram of a single high-level element XXX, it's possible to say the number of occurrences XXX given ddd.


Sample as a query


One way to calculate similarity between two data instances ddd and dsampled_{sample}dsample​ is to intersect their histograms H(d)H(d)H(d) and H(dsample)H(d_{sample})H(dsample​). This can be written as

sim(dsample,d)=min(H(dsample),H(d)))=(h(e1),⋯ ,h(en)∣h(x)=min⁡(h(x∣dsample),h(x∣d))).\text{sim}(d_{sample}, d) = \text{min}(H(d_{sample}), H(d))) = \left(h(e_1),\cdots,h(e_n) \mid h(x)=\min(h(x \mid d_{sample}), h(x \mid d)) \right).sim(dsample​,d)=min(H(dsample​),H(d)))=(h(e1​),⋯,h(en​)∣h(x)=min(h(x∣dsample​),h(x∣d))).

So, the histogram representation of data instances is an appropriate form for using different types of queries that is a useful feature for information retrieval systems.

5.3. Histogram model vs probability model


A common approach for probability estimation is based on the number of occurrences for a particular outcome divided by the number of all observed outcomes. In histogram model we deal with the number of occurrences directly (or using a normalized form as well) and we don't calculate any probabilities. As a result, operations over probabilities are not applied to histograms. Moreover, the histogram algebraic system works with vectors (histograms) as a single unit.

5.4. Histogram model vs fuzzy set model


The most obvious difference between these two models from a mathematical point of view is that a histogram is restricted by the the number of elements of a data instance, whereas a fuzzy set has no such limitations. For example, if we take a normalized histogram, sum of its elements can not more than 1, whereas for a fuzzy set, each of its element can be from 0 to 1.

As known, the fuzzy sets don't follow the law of contradiction and the law of excluded middle. In turn, the histogram model complies with the laws as shown below, respectively (over two operations: union and intersection):

H∩H‾=0,H \cap \overline{H} = \mathbf{0},H∩H=0,
H∪H‾=1.H \cup \overline{H} = \mathbf{1}.H∪H=1.

This feature is inherited from the sets.
